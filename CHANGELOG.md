# 版本记录

本项目在首个公开版本前使用过 2.x、1.0.1 和 1.0 等内部里程碑标签。这些标签从未对外分发，不代表已发布的 SemVer 序列；当前发布线从 `1.0.0-rc.1` 开始，当前开发候选为 `1.0.0-rc.3`。

## 1.0.0-rc.3

### 安全与可靠性

- 移除 `cookies` 权限和公共站点探测中的 Cookie 读取；登录态操作继续固定在目标站标签页上下文执行。
- 站点与恢复快照写入前增加存储配额预检，并将浏览器 quota 失败统一为稳定错误码 `storage_quota_exceeded`。
- 新增独立 runtime 消息契约，集中登记 43 个现有消息类型；业务分发前校验发送方、标识、数组数量、文本大小和对象结构，并以稳定错误码拒绝未知或不可信请求。
- 增加 Popup 顶部“一次授权”入口，合并请求当前收藏站点的精确 HTTPS Origin；获取 Key 自动补开的令牌页改为后台标签，不切走当前页面。

### 工程门禁

- 增加跨平台全量 JavaScript 语法检查命令。
- CI 增加覆盖率报告、Chromium 扩展 runtime smoke 和 Playwright UI gate。
- `v1.0.0-rc.2` 的源码标签和发布制品保持不可变，不会被本轮开发覆盖。

### 仓库整理

- 删除已退出运行路径的签到桥接模块、对应旧测试和一次性拆分脚本。
- 删除旧实施计划、UI 概念稿和重复的过程报告；当前说明集中到 README、架构决策和 RC 资料。
- 清理本机上传、构建和 Playwright 临时产物；保留 rc.2 不可变发布物。

## 1.0.0-rc.2（候选准备中）

### 加固与验证

- 延续 rc.1 的 Origin、Key、导入恢复、余额停止与重启恢复边界，并将本轮修复统一纳入当前候选。
- 增加本机恢复快照维护、孤立 HTTPS 授权清理和并发/竞态回归覆盖。
- 增加隔离 Edge/Chromium runtime smoke 与 Playwright UI gate，覆盖空状态、筛选排序、键盘、主题、缩放和大数量站点。
- 发布资料、版本元数据和确定性制品命名统一切换到 rc.2；rc.1 文件和标签作为不可变历史证据保留。

### 发布门禁

- 自动冒烟必须通过，且 `npm test`、构建、包白名单和 UI 门禁必须在最终 rc.2 代码状态重跑。
- 单名测试者必须使用专用测试账号和 Key，并在 Chrome Stable 与 Edge Stable 上完成交叉浏览器验收（推荐五天窗口）。
- P0、P1 和阻断型 P2 均为零后，维护者才能签署发布。

## 1.0.0-rc.1（待发布）

### 新增

- Popup 与 Options 采用统一设计系统、响应式布局和系统深色模式。
- Popup 保留全部站点的独立滚动列表，提供健康、分类、标签和搜索筛选。
- 批量余额刷新提供协作式停止与继续，保留已完成结果和未处理队列。
- Options 按站点管理、导入与备份、诊断与权限划分工作区。
- 导入必须先预览新增、更新和跳过数量。
- 提供手动复制的脱敏诊断，不包含遥测或自动上传。
- 提供无账号、无 Key 的真实扩展上下文冒烟门禁。

### 安全与正确性

- 站点权限必须在用户手势仍有效的前台点击处理器中请求。
- 批量刷新即使有站点拒绝授权，也继续处理已授权站点并将未授权项记为跳过。
- 完整 Key 不进入普通列表、诊断复制、日志或验收材料。
- Origin 保留显式端口；站点 URL 保存前移除 query 和 hash。
- 完整导出、替换导入、恢复和清理均要求明确确认。

### 发布门禁

- 隔离 Edge/Chromium 扩展上下文自动冒烟必须通过；Chrome Stable 与 Edge Stable 必须另行完成双人真实验收。
- 两名测试者必须使用相互独立的专用测试账号和 Key，并完成五天交叉浏览器验收。
- P0、P1 和阻断型 P2 均为零后，维护者才能签署发布。
