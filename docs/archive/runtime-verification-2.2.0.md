## 操作与取证方法

1. 打开 `chrome://extensions` 或 `edge://extensions` 并启用开发者模式。在两个扩展各自的卡片中点击「service worker」/「检查视图」链接，分别打开 DevTools；重新加载扩展后观察 Console，并保留错误或无错误状态的截图。
2. 要确认签到侧存储，请在「公益站签到」的 service worker DevTools Console 执行 `chrome.storage.local.get(null).then(console.log)`，在输出的站点集合中查找测试域名。只使用测试站点，不在验收记录中粘贴真实 Key 或完整存储快照。
3. Chromium 默认不会展示跨扩展消息正文。检查直接同步 payload 时，可在发送端调用 `sendToExtension` 前，或接收端 `importSitesFromHub` 处理入口临时加入仅用于本地验收的 `console.debug`，重新加载两个扩展后用测试数据触发一次同步。观察 `importSitesFromHub` payload，确认没有 `keys`、`apiKey`、`sk-` 或 URL 深路径；记录结果后删除临时日志并再次重新加载，不能把临时调试改动带入发布版本。
4. 若签到扩展没有稳定的逐站失败注入方式，不伪造「部分失败」结果：以自动化的部分回读测试作为补充证据；真实环境至少验证全失败、失败状态持久化和重试恢复，并在观察结果中注明部分失败未在真实环境注入。
