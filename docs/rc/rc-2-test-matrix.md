# 1.0.0-rc.2 双人测试矩阵

两名测试者必须使用彼此独立的专用账号、Key、浏览器 profile 和测试数据。任何生产凭据都不在本矩阵范围内。

## 环境分工

| 测试者 | 主浏览器 | 交叉冒烟 | 主题与缩放 | 周期 |
| --- | --- | --- | --- | --- |
| A | Chrome Stable，连续主验收 | Edge Stable，干净 profile | 深色，125% | 5 天 |
| B | Edge Stable，连续主验收 | Chrome Stable，干净 profile | 浅色，150% | 5 天 |

两人都应额外在 100% 缩放完成一次 Popup 和 Options 基础检查。浏览器版本必须记录具体版本号，不能只写“最新版”。

## 两人都必须覆盖

- [ ] 全新加载版本化 ZIP，而不是源码目录或旧 RC 覆盖目录。
- [ ] 权限允许、拒绝和再次授权。
- [ ] 单站及批量余额，包含手动停止、继续和浏览器重启恢复。
- [ ] Key 导入、创建前取消、明确确认后创建和分离复制。
- [ ] 导入预览中的新增、更新、跳过以及取消零修改。
- [ ] 脱敏导出、完整导出确认、替换快照和恢复。
- [ ] 手动复制脱敏诊断并检查敏感字段缺失。
- [ ] Popup、Options、菜单、drawer、dialog 的键盘和缩放。

## 交叉冒烟最低范围

交叉浏览器必须使用全新 profile，并至少验证：

- [ ] 扩展加载和 service worker 启动无错误。
- [ ] Popup、Options 和三个 Options 工作区可打开。
- [ ] 单站权限拒绝后可再次授权。
- [ ] 一个专用测试站完成收藏、Key 和余额主流程。
- [ ] 批量刷新可停止并继续。
- [ ] 导入取消不修改存储，脱敏诊断不含敏感值。
