# 1.0.0-rc.2 单人双浏览器测试矩阵

当前 RC 按**单人验收**执行。同一名测试者必须在 **Chrome Stable** 与 **Edge Stable** 上各用**独立浏览器 profile**、**专用测试账号/Key**（可同站不同账号，但不得使用生产凭据）完成主验收；不得与生产浏览器 profile 混用。

历史 rc.1 文档中的双人流程仅作归档参考，不适用于本候选。

## 环境分工

| 角色 | 主浏览器 | 交叉冒烟 | 主题与缩放 | 周期 |
| --- | --- | --- | --- | --- |
| 唯一测试者 | Chrome Stable，连续主验收 | Edge Stable，干净 profile 冒烟 | 深色，125% | 5 天 |
| 唯一测试者 | Edge Stable，连续主验收 | Chrome Stable，干净 profile 冒烟 | 浅色，150% | 5 天 |

同一人在五天内完成上表两行；两套 profile 不得共享扩展本地存储。浏览器版本必须记录完整版本号，不能只写“最新版”。另在 100% 缩放各做一次 Popup/Options 基础检查。

## 五天安排

| 天 | Chrome 主线 | Edge 主线 | 当日证据 |
| --- | --- | --- | --- |
| 1 | 全新安装、service worker、Popup/Options、允许与拒绝权限 | 同左；与 Chrome 交换允许/拒绝顺序 | 版本、控制台结果、脱敏步骤 |
| 2 | 收藏、识别、Origin/显式端口、导入已有 Key、取消/确认创建 | 同左；使用本浏览器独立 profile 与测试数据 | 不含完整值的结果摘要 |
| 3 | 单站余额、批量刷新、停止、继续、扩展重新加载恢复 | 同左，并增加浏览器完全重启恢复 | 批量计数和稳定错误码 |
| 4 | 导入预览、取消零修改、合并、替换、恢复、脱敏/完整导出 | 同左；交换文件与粘贴输入路径 | 仅记录计数，不附导出文件 |
| 5 | 深色 125%、键盘与回归；Edge 干净 profile 交叉冒烟 | 浅色 150%、键盘与回归；Chrome 干净 profile 交叉冒烟 | 最终问题状态与签署 |

若时间压缩，至少保证：**两天完整主流程**（Chrome 一天 + Edge 一天）+ 各浏览器一次交叉冒烟；五天为推荐完整窗口。

## 必须覆盖

- [ ] 全新加载版本化 ZIP，而不是源码目录或旧 RC 覆盖目录。
- [ ] 权限允许、拒绝和再次授权（Chrome 与 Edge 各至少一轮）。
- [ ] 单站及批量余额，包含手动停止、继续和浏览器重启恢复。
- [ ] Key 导入、创建前取消、明确确认后创建和分离复制。
- [ ] 导入预览中的新增、更新、跳过以及取消零修改。
- [ ] 脱敏导出、完整导出确认、替换快照和恢复。
- [ ] 手动复制脱敏诊断并检查敏感字段缺失。
- [ ] Popup、Options、菜单、drawer、dialog 的键盘和缩放。

## 交叉冒烟最低范围

交叉浏览器必须使用全新 profile，并至少验证：

- [ ] 扩展加载和 service worker 启动无错误。
- [ ] Popup、Options 和三个 Options 工作区可打开。
- [ ] 单站权限拒绝后可再次授权。
- [ ] 一个专用测试站完成收藏、Key 和余额主流程。
- [ ] 批量刷新可停止并继续。
- [ ] 导入取消不修改存储，脱敏诊断不含敏感值。
