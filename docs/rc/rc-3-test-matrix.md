# 1.0.0-rc.3 单人双浏览器测试矩阵

当前 RC 由一名可信测试者在 Chrome Stable 与 Edge Stable 上完成。两个浏览器必须使用独立 profile 和专用测试账号/Key，不得加载日常源码目录、复用旧 RC 解压目录或使用生产凭据。

## 环境记录

| 浏览器 | 完整版本 | Profile 是否全新 | ZIP SHA-256 已核对 | 主题/缩放 | 结果 |
| --- | --- | --- | --- | --- | --- |
| Chrome Stable |  |  |  | 深色 / 125% |  |
| Edge Stable |  |  |  | 浅色 / 150% |  |

两套 profile 不得共享扩展存储。另在两个浏览器的 100% 缩放各做一次 Popup、Options 和控制台基础检查。

## 推荐五天安排

| 天 | Chrome 主线 | Edge 主线 | 脱敏证据 |
| --- | --- | --- | --- |
| 1 | ZIP 校验、全新加载、权限允许/拒绝、rc.3 权限与消息边界 | 同左，交换权限顺序 | 版本、摘要、控制台结论 |
| 2 | 收藏、识别、显式端口、导入已有 Key、取消/确认创建 | 使用独立账号重复主线 | 仅记录状态和计数 |
| 3 | 单站/批量余额、停止、继续、扩展重载和浏览器重启 | 同左，交换故障顺序 | run 状态和稳定错误码 |
| 4 | 导入预览、合并、替换、恢复、配额失败、脱敏/完整导出 | 交换文件与粘贴路径 | 不附导出文件或存储 |
| 5 | 深色、125%、键盘与 Edge 交叉冒烟 | 浅色、150%、键盘与 Chrome 交叉冒烟 | 台账终态和签署 |

时间压缩时最低要求是 Chrome 和 Edge 各完成一次完整主流程，并各使用一个全新 profile 完成交叉冒烟。五天窗口用于覆盖重启恢复、凭据清理和复验。

## rc.3 专项

- [ ] 两个浏览器的扩展详情页均不显示 `cookies` 权限。
- [ ] 公共识别可用；已登录余额/Key 操作仍正确使用目标站标签页，不因移除 Cookie 权限退化。
- [ ] 使用测试环境触发配额失败时返回 `storage_quota_exceeded`，现有站点与快照保持完整。
- [ ] Popup、Options 的既有消息操作均成功；service worker 无 `unknown_message` 误拒绝。
- [ ] 开发者控制台构造未知消息时返回 `unknown_message`，构造超限消息时返回稳定拒绝结果。

## 主流程

- [ ] 权限拒绝后不访问站点，可从下一次前台点击重新授权。
- [ ] 同 hostname 不同 HTTPS 端口保持独立站点、权限、Key 和余额。
- [ ] 已有 Key 只导入/复用；空列表取消不创建，确认后最多创建一把。
- [ ] 单站和批量余额可处理成功、未登录、权限拒绝和超时。
- [ ] 批量停止后不开始下一站，继续时只处理 pending 项。
- [ ] 重载扩展和重启浏览器后能恢复中断状态。
- [ ] 导入必须先预览；替换前快照、恢复、清理和配额保护可用。
- [ ] 脱敏导出/诊断无敏感值，完整导出只在明确确认后生成。

## 交叉冒烟

- [ ] 从全新目录加载版本化 ZIP，service worker、Popup、Options 和三个工作区无错误。
- [ ] 一个专用测试站完成授权、收藏、Key 和余额主流程。
- [ ] 批量刷新可停止并继续，导入取消不修改存储。
- [ ] Popup/Options 无横向溢出，键盘可完成菜单和 drawer 主路径。
- [ ] 卸载扩展后清除 profile 数据并撤销测试 Key。
