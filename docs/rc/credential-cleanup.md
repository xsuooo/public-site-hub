# RC 测试凭据生命周期与清理

本流程适用于每个 RC，包括通过、失败和撤回的候选版本。仓库只记录清理状态，不记录任何凭据、账号标识或真实站点清单。

## 验收前

- [ ] 测试者 A、B 分别创建独立专用账号和 Key，不共享账号或 Key。
- [ ] 凭据只授予完成 RC 测试所需的最小能力、额度和有效期。
- [ ] 不从生产环境复制 Key、Cookie、导出文件或浏览器 profile。
- [ ] 每名测试者使用独立专用浏览器 profile。
- [ ] 约定凭据撤销责任人和最迟清理日期。

## 验收期间

- [ ] 完整 Key 不进入问题台账、截图、诊断、日志或聊天记录。
- [ ] 完整导出和恢复快照只保存在测试者本机受控位置。
- [ ] 剪贴板中的 Key 使用完毕后由其他非敏感文本覆盖。
- [ ] 不把测试浏览器 profile、下载目录或扩展存储提交到仓库。
- [ ] 发生疑似泄露时立即撤销相关 Key，并按 P0 处理。

## 验收结束后立即执行

- [ ] 撤销测试者 A 的全部 RC Key。
- [ ] 撤销测试者 B 的全部 RC Key。
- [ ] 清除两个测试 profile 中的扩展本地数据。
- [ ] 删除本机完整导出、恢复快照、诊断临时文本和测试下载。
- [ ] 清空或覆盖仍可能包含 Key 的剪贴板内容。
- [ ] 检查仓库、问题记录和截图，确认没有敏感值残留。

## 账号最长保留期

专用账号可在 RC 结束后最多保留七天用于复测，但其 Key 必须先立即撤销。七天内没有复测需求时提前删除或重置；到期必须删除账号或重置为不可复用状态。
