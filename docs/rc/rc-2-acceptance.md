## 权限与 Origin

- [ ] 单站权限由前台用户点击直接请求，没有 `This function must be called during a user gesture` 错误。
- [ ] 收藏、识别、重新识别、余额和 Key 入口只请求当前操作 Origin。
- [ ] 拒绝授权后不执行该站的网络、Cookie 或页面脚本操作，并可再次从用户手势重试。
- [ ] 批量刷新中拒绝一个未授权站点后，已授权站点继续处理；未授权项记为跳过，不中断整批。
- [ ] 同域不同账号标签页不会串用；跨站导航失败闭合。
- [ ] 显式端口测试 Origin，例如 `https://example.invalid:8443`，在权限、页面、会话、存储和请求中保持一致。

## Key 安全与操作

- [ ] 空值、截断值和掩码值不能被当作完整 Key 保存、复制或设为默认。
- [ ] 页面示例文本不会被自动保存为 Key。
- [ ] 已存在完整 Key 时，获取流程只导入或复用，不创建重复 Key。
- [ ] 兼容测试站远端令牌列表为空时，取消确认不会创建；明确确认后最多创建一把测试 Key。
- [ ] 无法确认登录、站点类型或令牌列表时不会猜测性创建。
- [ ] 普通列表和诊断界面只显示 Key 名称、状态或尾号；完整值不进入 DOM 文本、截图、日志和诊断复制。
- [ ] “复制接口地址”只有 Origin 加 `/v1`；“复制 Key”只有完整 Key，两个动作不会组合数据。

## 余额、停止与恢复

- [ ] 单站余额成功、未登录、类型错误、权限拒绝和超时均有稳定状态与可执行下一步。
- [ ] 批量刷新显示进度，并可手动请求停止。
- [ ] 停止为协作式：当前站保存后不再开始下一站；已完成结果保留，未处理项不记为失败。
- [ ] 停止后可以继续剩余队列，不重复处理已完成项。
- [ ] 刷新期间重新加载扩展或重启浏览器后，能识别中断并继续剩余站点。
- [ ] 过期事件或旧 runId 不会覆盖新一轮任务状态。

## 导入、导出与恢复

- [ ] 所有导入必须先预览当前总数、传入总数、新增、更新和跳过数量。
- [ ] 取消预览、替换确认或文件选择不会修改存储。
- [ ] 合并导入只新增或更新预览中的有效项。
- [ ] 替换导入先创建本机恢复快照，并显示完整 Key 风险。
- [ ] 可以恢复最近快照，也可以显式清理快照。
- [ ] 脱敏导出不能恢复完整凭据；再次导入时有明确提示。
- [ ] 完整导出要求危险确认并显示站点/Key 数量。
- [ ] 含 query/hash 的 URL 不进入存储、导出、诊断或问题材料。
