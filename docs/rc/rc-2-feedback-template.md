# RC 问题反馈模板

每个独立问题复制下面模板，并作为一行摘要加到 `rc-2-findings.md`。问题材料必须先脱敏；不确定是否敏感时，不要附上原始值。

## 严重级别

- **P0**：敏感数据泄露、数据丢失、操作错站或不可逆破坏。
- **P1**：主流程不可用、权限死循环、批量任务卡死，或导入导出无法恢复。
- **P2**：支持范围内稳定复现并明显影响工作流、正确性或可读性。
- **P3**：轻微视觉或便利性问题，有明确规避方式。

```text
操作系统：

实际结果：

```

## 提交前脱敏检查

- [ ] 没有完整或可恢复的 Key。
- [ ] 没有 Cookie、Authorization header、账号邮箱或令牌编号。
- [ ] 没有完整存储、完整导出、恢复快照或测试下载文件。
- [ ] 没有真实站点清单、私人备注或可识别账号的信息。
- [ ] URL 已去掉 query/hash；不需要域名时改用 `example.invalid`。
- [ ] 截图已遮住地址栏、扩展 ID、通知、下载路径和其他无关私人内容。
- [ ] 诊断由“复制脱敏诊断”生成，并再次人工检查。

禁止以聊天记录代替问题台账。发现 P0/P1 时应先停止使用该 RC，再记录和通知维护者。
