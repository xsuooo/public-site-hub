# 公益站收藏 UI 重构概念规格

日期：2026-07-17
状态：已批准并完成生产实现；当前作为 1.0 RC 的设计基线
范围：popup、options 和共享 UI

## 1. 目标

在不改变现有存储格式、权限模型、导入导出格式和核心业务语义的前提下，把扩展重构为克制、可信、高效的工具型界面。

`1.0` 尚未正式分发，因此本次 UI 重构属于首发门禁。概念稿已批准并落实到生产界面；发布前仍须完成自动化、Chrome 和 Edge 真实验收。

## 2. 已确认的产品边界

- 保持名称“公益站收藏”和现有紫色书签星标图标，不做品牌重塑。
- 保持原生 HTML、CSS、JavaScript，不引入前端框架或运行时依赖。
- 允许拆分 `popup.js`、`options.js` 的控制器、状态和视图代码。
- `shared-ui.css` 只保留 token 和共享组件；新增 `popup.css`、`options.css`，移除页面内联 `<style>` 与 `style=""`。
- 深色模式继续跟随系统，不增加主题切换器。
- 不引入远程字体、远程图标、分析 SDK 或崩溃遥测。
- 完整 Key 默认掩码，不进入普通列表 DOM 文本、截图、日志或诊断复制内容。
- 继续接受旧链接 `options.html#edit=<siteId>`，并可规范化到新的工作区 hash。
- popup 保持全部筛选结果可滚动查看，不分页；按最多约 100 个站点设计。

## 3. 视觉方向

### 3.1 原则

- 清爽、可信、低装饰；信息层级优先于渐变、阴影和卡片堆叠。
- 紫色只用于主操作、选中和焦点；黄色只用于品牌图标星标。
- 成功、警告、失败都有文字，不只依赖颜色。
- 普通面板以边框分区；只有 popover、drawer、dialog 使用明显阴影。
- 控件文字、状态、域名、Key 尾号都必须清晰可读，禁止 10px 正文和浏览器默认控件排版。

### 3.2 核心颜色

| Token | 浅色 | 深色 | 用途 |
|---|---|---|---|
| `--canvas` | `#F5F6FA` | `#0F1117` | 页面画布 |
| `--surface` | `#FFFFFF` | `#171A23` | 主表面 |
| `--surface-subtle` | `#F8FAFC` | `#1E2230` | 表头、次级区块 |
| `--text` | `#111827` | `#F4F6FB` | 主文字 |
| `--text-secondary` | `#4B5563` | `#C7CEDB` | 次级文字 |
| `--text-muted` | `#667085` | `#A9B2C3` | 元数据 |
| `--border` | `#D8DEE9` | `#343B4C` | 边框 |
| `--primary` | `#4F46E5` | `#958EFF` | 主操作、选中、焦点 |
| `--primary-soft` | `#EEF2FF` | `#29264B` | 选中背景 |
| `--success` | `#167647` | `#5DD58C` | 成功 |
| `--warning` | `#8A4B0F` | `#F2C15B` | 警告 |
| `--danger` | `#B42318` | `#FF8195` | 失败和危险 |

### 3.3 排版和几何

- UI 字体：英文优先 `Segoe UI Variable Text / Segoe UI`，中文优先本机 `Noto Sans SC`，缺失时回退 `Microsoft YaHei UI / Microsoft YaHei / PingFang SC`；禁止远程字体。
- 标题优先 `Segoe UI Variable Display`；中文继续使用上述无衬线回退，中文标题不得使用负字距。
- 域名、schema、完整 Key 值和诊断字段使用 `Cascadia Mono / Cascadia Code / SFMono-Regular / Consolas` 本地等宽栈，并关闭连字。
- 字号只使用 11、12、13、14、16、18、22px；字重只使用真实的 400、500、600、700，设置 `font-synthesis: none`，禁止 650、750、800 合成字重。
- 生产实现必须删除 popup 中 `Microsoft YaHei UI` 优先于 `Segoe UI` 的页面级覆盖；所有页面统一继承共享字体 token。
- 生产 popup 现有 10px 元数据、状态与 Key 摘要统一提高到至少 11px；站点名使用 14px/600，主按钮 13px/600，普通按钮 11px/500。
- Key 名称继续使用 UI 字体；完整 Key 值使用等宽字体。紧凑卡片中的 Key 尾号使用 UI 字体、500 字重和等宽数字，避免和中文元数据混排时产生字重、基线跳变。
- 4px 间距基准：2、4、6、8、12、16、20、24、32、40px。
- 控件圆角 8px，卡片/popover 12px，drawer/dialog 16px，pill 999px。
- 动效只用于 drawer、dialog、popover、toast，120–180ms；遵循 `prefers-reduced-motion`。

## 4. Popup 概念

### 4.1 容器

- 目标尺寸：420×640 CSS px；概念图可以使用 440×680 画布留出安全边缘。
- 根布局使用固定头部与唯一滚动区；页面本身不滚动，只有站点列表滚动。
- 较矮窗口只收紧间距和隐藏品牌副标题，不隐藏收藏、搜索或筛选。

### 4.2 从上到下的信息架构

1. 紧凑品牌栏
   - Logo、`公益站收藏`
   - 一个明确的 `管理收藏` 入口
2. 主操作行
   - split button：`收藏当前页` + 分类下拉
   - `刷新余额`
   - 批量刷新进行时，列表顶部 progress strip 必须提供文字 `停止`；停止为协作式操作，当前站点完成并保存后不再开始下一站，已完成结果不得回滚，未处理站点不得计为失败。
3. 单行健康筛选
   - `全部 40 · 正常 22 · 需处理 14 · 失败 4`
   - 每项均可点击，不能再使用三块大统计卡
4. 列表工具
   - 分类分段：`全部 / 公益 / 中转`
   - `标签` popover 按钮，显示当前选择
   - 搜索：`搜索名称、域名、备注或标签`
5. 站点列表
   - 全部筛选结果保留在一个滚动列表中，无分页

### 4.3 站点卡片

- 第一行：站点名称、文字健康状态、“更多”按钮。
- 第二行：完整 Origin（显式端口必须可见）、分类和最多两个标签。
- 数据行：缓存余额、相对更新时间、Key 状态；打开 popup 不自动联网。
- 常驻操作：`打开站点`、`复制接口地址`、`复制 Key`（仅完整 Key 存在时）。
- 状态动作：`获取 Key`、`刷新余额`、`重试并授权`、`再查余额`，只在适用状态出现。
- `编辑`、`重新识别`、`删除`进入“更多”菜单，不再通过展开卡片形成按钮墙。
- 残缺或掩码 Key 只能显示“待修复/删除/诊断”，不能呈现可复制或设默认的假动作。
- 复制成功优先在原按钮短暂显示 `已复制`；失败在原位置显示原因和重试。

### 4.4 必须出现的概念状态

- 混合站点：正常、余额失败、无 Key/Key 可选、显式 `:8443` 端口。
- 空状态：`还没有站点`，提供 `收藏当前页` 下一步，不出现全选或空白按钮。
- 局部刷新进度：保留当前列表，只显示非阻塞 progress strip。
- 深色模式：信息密度、边框和状态对比保持一致。

## 5. Options 概念

### 5.1 应用外壳

- 桌面最大内容宽度约 1220px。
- 宽屏采用 200px 左侧工作区导航 + 主内容；导航 sticky。
- 工作区：`站点管理`、`导入与备份`、`诊断与权限`。
- `≤860px` 导航切换为顶部三项导航，内容单栏。
- 工作区 hash：`#view=sites`、`#view=import`、`#view=diagnostics`、`#view=sites&edit=<siteId>`。
- 旧 `#edit=<siteId>` 必须继续打开站点管理和对应 drawer。

### 5.2 站点管理

- 页头：工作区标题、站点数量、主操作 `添加站点`、全局 `刷新全部余额`。
- 工具栏：搜索、分类、标签、临时排序（名称/最近更新/健康状态）。
- `≥1100px` 使用高密度表格：选择、名称/域名、分类/标签、余额、Key、健康、操作。
- `861–1099px` 与窄屏使用响应式卡片；100 个筛选结果仍不分页。
- 有选择时出现吸顶批量栏：`已选 N`、`全选当前`、`取消选择`、`删除所选`。
- 无数据或无匹配时不显示选择框和批量操作。
- 添加与编辑统一使用右侧 drawer；进入 `≤860px` 单栏布局后 drawer 全宽。
- drawer 字段与现有站点数据保持一致；Key 默认掩码，残缺 Key 不可复制或设默认。

### 5.3 导入与备份

- 宽屏使用 1.2fr / 0.8fr 分栏。
- 左侧：文件/粘贴输入、格式说明、预览结果。
- 所有导入都必须先预览新增、更新、跳过、当前总数和传入总数。
- `合并导入`为普通主动作；`替换导入`为危险次级动作并说明会创建恢复快照。
- 右侧：`导出（脱敏）`、`导出（含完整 Key）`、恢复快照、清理快照。
- 完整导出、替换、恢复、清理都使用统一 `<dialog>`；不得并列成相同视觉权重的按钮墙。

### 5.4 诊断与权限

- 默认展示用户可理解的摘要、问题列表和可执行动作。
- 操作包括：`授权未授权站`、`重试余额失败`、`打开失败站`、`打开未登录站`、`刷新`。
- `技术详情`使用折叠区，包含版本、schema、迁移时间、scope 和本地计数。
- 提供 `复制脱敏诊断`；完整 Key 和敏感 URL 不得进入复制内容。
- `自动创建无限额度且永不过期 Key`移动到高级设置，默认关闭并附风险说明。

## 6. 共享组件边界

- button：primary、secondary、ghost、danger-secondary、danger、icon。
- field：label、input/select/textarea、hint、error。
- segmented、status-pill、notice、progress-strip、toast、empty-state。
- popover/menu、dialog、drawer。
- key-item、data-table、responsive-site-card、bulk-toolbar。
- 状态优先使用 `data-state`、`data-tone`、`aria-busy`、`aria-current`、`aria-pressed`，脚本不拼接一次性视觉样式。
- 本地内联 SVG 只用于清晰的导航与操作图标；关键动作仍有文字或可访问名称。

## 7. 响应式、深色和无障碍门禁

- popup：420×640，100% 和 125% 缩放；顶部核心操作固定，列表独立滚动。
- options：1280×900 与 768×1024；100% 和 125% 缩放；不得横向滚动。
- `>1100px` 表格；`861–1099px` 侧栏 + 卡片；`≤860px` 顶部导航 + 单栏及全宽 drawer；`≤640px` 操作组允许纵向堆叠。
- 浅色与深色分别检查正文、状态、焦点和 dialog/drawer。
- 所有控件必须有 accessible name，表单使用真实 `<label>`。
- Tab/Shift+Tab、Enter/Space 可用；Esc 关闭菜单、drawer、dialog 和 Key 临时揭示，并返回触发器焦点。
- 正文对比度至少 4.5:1，大字至少 3:1。

## 8. 概念图生成清单

每张图都是完整、可实现的产品 UI，不是宣传页、设备样机或静态截图替代品。不得发明新功能、远程服务、营销指标或无关导航。

### C1：Popup 混合状态（浅色）

- 画布：440×680 竖向应用屏幕。
- 显示紧凑品牌栏、split 收藏按钮、单行健康筛选、分类/标签/搜索工具和三张站点卡。
- 三张卡分别表现：正常 + 可复制 Key、显式 `:8443`；余额失败 + `重试并授权`；无 Key + `获取 Key`。
- 中文文案必须使用本规格中的术语；无设备外壳、无装饰插画、无大面积渐变。

### C2：Popup 空状态与深色模式

- 同一 420×640 结构的深色版本。
- 空状态包含明确下一步；隐藏无意义筛选计数和批量控件。
- 展示清晰焦点环；标签 popover 在混合状态交互中单独验收。

### C3：站点管理（桌面浅色）

- 画布：1280×900。
- 左侧三工作区导航，右侧高密度站点表格，混合健康状态与吸顶批量栏。
- 打开一个编辑 drawer，展示字段和掩码 Key；背景列表仍可理解但不可交互。
- 表格不能变成卡片网格，不能出现营销式统计大卡。

### C4：导入与备份（桌面浅色）

- 画布：1280×900。
- 显示文件/粘贴输入、导入预览、新增/更新/跳过结果、合并与危险替换的层级。
- 右侧显示脱敏导出、完整导出与三份恢复快照；打开替换确认 dialog。

### C5：诊断与权限（桌面深色）

- 画布：1536×1024。
- 用户摘要、问题列表和修复动作优先；技术详情折叠；高级无限 Key 偏好带风险说明。
- 显示 `复制脱敏诊断`，不显示任何完整 Key。

### C6：Options 窄屏响应式

- 画布：768×1024。
- 顶部三工作区导航，站点列表卡片化，添加/编辑 drawer 变为全宽面板。
- 无横向滚动、表格硬压缩、截断关键按钮或浮动按钮墙。

## 9. 自动化概念验收基线

- 增加 `@playwright/test` 开发依赖，不进入 `dist/`。
- 合成场景：empty、single、mixed、hundred；域名使用 `.invalid`，Key 使用明显不可用测试值。
- Playwright 从 service worker URL 动态取得 extension ID，隔离 storage 与临时 profile，`workers: 1`。
- 自动检查结构、可见性、hash、键盘、溢出、控制台、深色和 125% 缩放；截图只作人工证据，不做跨机器像素硬比较。
- Chrome 完成全矩阵；Edge 完成全新安装、权限拒绝/重试、一个 NewAPI 完整流程和一个非 NewAPI 防误创建流程。

## 10. 概念批准门槛

进入编码前必须确认：

1. popup 的列表优先层级正确，健康统计和标签不再抢占高度；
2. 站点卡常驻动作与状态动作清晰，且没有按钮墙；
3. options 的三个工作区、表格、drawer、导入预览和诊断层级完整；
4. 浅色、深色和 768px 响应式属于同一设计系统；
5. 可见文案、状态、安全边界和旧深链接没有被概念图改变；
6. 概念实际可用 HTML/CSS/原生 JavaScript忠实实现。

本文件已升级为生产设计规格；实现不得擅自改变已批准布局、文案、组件模型和视觉 token。后续可见偏差必须在 RC 问题台账中记录并按严重级别处理。
