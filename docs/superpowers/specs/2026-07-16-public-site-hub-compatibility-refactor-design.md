# 公益站收藏兼容优先重构设计

## 背景

当前项目是无构建步骤的 Manifest V3 浏览器扩展，使用经典脚本和 `importScripts` 共享全局 API。现有 Node 测试共 183 项并全部通过，但真实 Chromium 验收仍未完成，而且已复现多项现有测试未覆盖的运行时缺陷。

本次重构保持现有界面、站点数据字段、导入导出格式和主要交互兼容。目标是先修复可复现的正确性与发布问题，再收敛重复代码和运行时边界，不在同一批次迁移到 ESM、引入前端框架或重做视觉设计。

## 目标

1. 在探测、权限、标签页、会话和调用地址中完整保留 HTTPS Origin，包括显式端口。
2. 消除偏好、备份和替换导入的读改写竞态。
3. 让过期恢复快照及其中的完整 API Key 真正从本地存储删除。
4. 统一 popup 与 options 的公共 UI 行为，修复剪贴板假成功和掩码 Key 可操作问题。
5. 让并发余额任务明确表达活动范围，不再静默忽略第二个请求的范围。
6. 建立可重复的发布包白名单与图标校验，避免发布开发文件、截图和空白图标。
7. 用行为测试和 Chromium 冒烟验证补足当前以源码正则为主的测试缺口。

## 非目标

- 不改变 `sites`、`prefs`、`siteBackups` 等现有存储键。
- 不删除旧站点记录中的签到兼容字段。
- 不改变完整备份、脱敏备份和签到格式的外部 JSON 结构。
- 不引入 React、TypeScript、打包器或第三方运行时依赖。
- 不全面重写 `background.js`、`popup.js` 或 `options.js`。
- 不把真实站点凭据写入测试、日志、截图或发布包。

## 方案选择

### 采用：兼容优先的分层重构

按“地址语义、存储事务、共享 UI、发布门禁”四个独立边界逐步修复。每个缺陷先增加失败测试，再进行最小实现，确保现有行为持续兼容。

### 暂不采用：立即深拆全部大文件

进一步拆分 `background.js`、`popup.js` 和 `options.js` 能降低长期维护成本，但会同时改变大量全局加载关系，增加回归范围。本轮只抽取已经在两个页面重复、且与已确认缺陷直接相关的公共 UI 代码。

### 暂不采用：整体迁移 ESM 和构建工具

ESM 能提供明确依赖和静态导入，但会改变 service worker、popup、options、测试和发布方式。本轮先建立稳定边界和发布门禁，为后续迁移提供可信基线。

## 架构设计

### 1. 统一站点地址语义

`domain` 继续保存纯 hostname，保持现有数据兼容；所有需要网络或浏览器权限的代码改为通过统一 helper 从 `baseUrl`、`pageUrl`、`domain` 推导规范 HTTPS Origin。

新增或迁移到 `site-utils.js` 的公共能力包括：

- 取得站点规范 Origin，并保留显式端口。
- 生成 `Origin + /v1` 的调用地址。
- 生成权限 pattern `Origin + /*`。
- 对预期 Origin 和实际标签页 Origin 做严格比较。

`detect.js` 的网络探测接收 Origin，不再只接收 hostname。`permissions.js` 的未授权站点记录保留 Origin，批量授权不得从 `site.domain` 重新构造默认 443 地址。`bridge.js` 中与收藏功能仍有关联的调用地址格式化迁移到 `site-utils.js`。

### 2. 存储事务边界

保留现有站点 mutation queue，并增加明确的偏好和备份 mutation queue。所有针对同一存储键的读改写操作必须通过对应队列执行。

新增原子服务 `replaceSitesWithBackup`：

1. 进入站点 mutation queue。
2. 读取最新站点快照。
3. 在备份 queue 中保存该快照。
4. 写入替换后的站点列表。
5. 返回新列表和实际对应的备份信息。

这样可以保证恢复快照对应替换发生前的最后一次成功编辑。普通合并导入继续通过站点 mutation queue 执行，不建立替换快照。

`saveSites` 更新元数据时保留已有 `migratedAt`，只更新 `schemaVersion` 和 `updatedAt`。读取恢复快照时若发现过期项，必须把清理后的列表写回 `chrome.storage.local`；清空后允许移除或写入空的 `siteBackups`，但对外返回结构保持不变。

### 3. 共享 UI runtime

新增无第三方依赖的经典脚本模块 `ui-runtime.js`，供 popup 和 options 在各自主脚本前加载。它只负责两页共同的基础行为：

- 包装 `chrome.runtime.sendMessage`，统一返回 `{ ok, error, ... }`。
- HTML 和属性转义。
- 完整 API Key 判定。
- 剪贴板写入，并明确返回成功或失败结果。
- 可选的短延迟输入调度，避免每次按键同步重建完整列表。

页面仍保留各自的状态、文案和渲染函数。不会把 popup 与 options 合并为一个大型抽象层。

设置页只有完整 Key 才显示“复制”和“设默认”操作；掩码或残缺 Key 只显示诊断状态和删除入口。所有复制调用方只有在共享剪贴板函数返回成功后才能显示成功提示。

设置页搜索框和动态多选框补充可访问名称。现有视觉层级、卡片布局和按钮位置保持不变。

### 4. 余额任务范围

余额批量任务为当前运行记录规范 scope：

- `all`：刷新全部或继续中断任务。
- `failed`：只重试当前失败站点集合。
- `explicit`：显式站点 ID 集合。

完全相同 scope 的重复请求可以共享同一 Promise。不同 scope 的请求不得静默附着到现有任务；返回稳定的 `balance_refresh_busy` 错误，并包含活动 scope，UI 显示正在执行的真实任务。此次不建立持久化任务队列，避免用户一次点击积压多轮站点访问。

### 5. 休眠签到代码

`checkin-sync.js` 和跨扩展发现逻辑不再进入 popup、options 或 service worker 的实际加载图。旧数据字段和导入导出兼容仍保留在 `site-utils.js`、`storage.js` 与 `import-export.js`。

仍被收藏功能使用的通用 helper 先迁移到合适模块，再移除 `bridge.js` 的运行时引用。遗留模块测试可继续保留，但发布信心和覆盖率报告需区分“当前运行路径”和“遗留兼容模块”。

### 6. 发布包与图标

以 `icons/logo.svg` 和现有有效小图标为视觉来源，重新生成非透明的 128px PNG，并增加尺寸和非透明像素校验。

新增确定性的 `npm run build` 发布脚本，将 manifest 实际需要的 HTML、CSS、JS 和图标复制到干净的 `dist/` 目录；`npm run verify:package` 单独校验产物。发布包禁止包含：

- `uploads/`
- `tests/`
- `scripts/`
- `docs/`
- 未进入运行图的签到遗留模块
- 本地测试输出和临时文件

发布校验同时检查 manifest 与 HTML 引用的文件均存在。

## 数据流

1. popup/options 通过共享 UI runtime 发送结构化消息。
2. background 消息路由只负责参数校验和调用服务。
3. 地址 helper 生成规范 Origin，交给探测、权限、标签页和调用地址逻辑。
4. 所有站点、偏好和备份读改写进入各自事务队列。
5. background 返回明确结果、错误 code 和活动任务 scope。
6. UI 只根据真实结果更新本地状态和用户提示。

## 错误处理

- 关键写入失败必须返回稳定错误，不允许显示成功提示。
- 剪贴板失败保留失败状态，不得被调用方覆盖。
- 不同范围余额任务冲突返回 `balance_refresh_busy`，不伪装成已执行。
- 探测和授权若无法保留目标 Origin，应失败闭合，不回退到同 hostname 的默认端口。
- 过期备份清理失败时不删除当前站点数据，并在诊断或调用结果中暴露错误。
- 现有面向用户的中文错误文案尽量保持，新增 code 供测试和 UI 分支使用。

## 测试设计

所有生产代码改动遵循红—绿—重构流程。

### Node 行为测试

- 显式端口在探测请求、权限 pattern、打开站点和复制调用中保持一致。
- 并发偏好 patch 不互相覆盖。
- 替换导入与并发站点编辑产生正确的最新快照。
- `saveSites` 不删除 `migratedAt`。
- 过期备份在底层存储中被物理清除。
- 剪贴板拒绝时页面不显示成功。
- 掩码 Key 不能复制或设为默认。
- 全量余额与失败重试并发时返回明确 busy scope。
- 128px 图标有正确尺寸且包含非透明像素。
- 发布目录只包含白名单文件，所有运行时引用完整。

### Chromium 冒烟测试

- 加载 `dist/` 后 service worker 无启动错误。
- popup 和 options 能启动，控制台无未处理异常。
- 基础搜索、筛选、编辑入口和设置页导航可操作。
- 对不依赖真实账号的剪贴板失败、可访问名称和扩展资源加载路径进行自动验证。
- 需要真实登录站点或真实 Key 的项目继续记录到 `docs/runtime-verification-1.0.md`，执行时不得保存 Key 证据。

## 完成条件

- 所有新增回归测试先失败后通过。
- `npm test` 全部通过。
- 关键运行模块覆盖率不低于重构前，并新增缺陷路径覆盖。
- 全部 JavaScript 通过语法检查。
- 发布目录校验通过，128px 图标非透明，包中无截图和开发文件。
- Chromium 能加载发布目录，service worker、popup、options 无启动错误。
- 真实账号相关验收项若无法自动完成，必须明确保留为待人工验证，不能宣称发布门禁已全部完成。

## 后续工作

本轮完成并稳定后，可单独设计下一阶段：将 background 消息路由拆分为领域 handler，或将经典脚本渐进迁移到 ESM。这些工作不属于本次实现范围。
